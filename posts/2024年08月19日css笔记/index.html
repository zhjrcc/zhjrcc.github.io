盒模型

将页面的盒模型设为 border-box

:root {
  box-sizing: border-box;
}

*,
::before,
::after {
  box-sizing: inherit;
}

负外边距

正的外边距会将容器的边缘往里推，而负的外边距则会将边缘往外拉。
负外边距的具体行为取决于设置在元素的哪边。

控制溢出行为

overflow：visible hidden scroll auto。
visible,默认值，内容可见，就算溢出容器。
hidden，隐藏溢出的内容。
scroll，显示滚动条。
auto，如果内容溢出显示滚动条。

表格布局

容器使用display:table，主栏和侧边栏使用display:table-cell。 单元格间距使用border-spacing(margin在单元格上不生效)，使用负边距抵消单元格外边缘的边距。
table

Flexbox

将容纳器设定display:flex;设定容器内的各个元素的宽度占比以及外边距，子元素默认是等高的。
table

min-height/width和max-height/width

不要明确指定元素的高度，会导致更复杂的情况。

使用min-height和max-height指定最大值和最小值，而不明确指定高度。元素就能在界限内自动决定高度度。

垂直居中内容

vertical-align:middle,只对行内元素或者table-cell单元格有效，无法作用到块级元素中。

  1. 自然高度的容器：给容器加上相等的上下内边距让内容居中
  2. 不需要指定高度或者避免使用内边距：对容器使用 display: table-cell 和vertical-align: middle
  3. Flexbox居中内容
  4. 容器里面的内容只有一行文字：设置一个大的行高，让它等于理想的容器高度
  5. 容器和内容的高度都知道：将内容绝对定位
  6. 不知道内部元素的高度： 用绝对定位结合变形(transform)

外边距折叠

当顶部和底部的外边距相邻时，就会重叠，就产生了单个外边距，这种现象就是外边距折叠。（只有上下外边距会产生折叠，左右外边距不会折叠。）

外边距折叠的主要原因与包含文字的块之间的间隔相关，分别标出了标题(左图)和段落(右图)的外边距：

折叠外边距的大小等于相邻外边距中的最大值。

防止外边距折叠：

  1. 对容器使用 overflow: auto(或者非 visible 的值)，防止内部元素的外边距跟容器外部的外边距折叠。这种方式副作用最小。
  2. 在两个外边距之间加上边框或者内边距，防止它们折叠。
  3. 如果容器为浮动元素、内联块、绝对定位或固定定位时，外边距不会在它外面折叠。
  4. 当使用 Flexbox/Gird 布局时，弹性布局内的元素之间不会发生外边距折叠
  5. 当元素显示为 table-cell 时不具备外边距属性，因此它们不会折叠。

猫头鹰选择器

猫头鹰选择器：body * + * ，选择的是页面上所有具有同一父元素的非第一个子元素。通常只在有并列元素，或者有多列布局时这样使用。在某些场景会很有用，但是不要滥用。
